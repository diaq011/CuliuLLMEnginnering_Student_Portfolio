# Ballie：J人模拟器

### 本节课最适合先做的版本

本节课不建议实现桌面小组件、手机使用监测或完整监督系统，更适合先完成“**任务拆解与日程生成器**”。

建议本节课的 MVP 收缩为：

- 输入：任务清单、截止时间、可学习时段
- 输出：按天拆解的执行计划

### 建议的最小功能

1. 页面支持输入：
   - 多个任务
   - 各自截止时间
   - 每天可学习时段
2. 后端调用模型生成日计划
3. 输出采用固定结构，例如：
   - `daily_plan`
   - `priority_reason`
   - `estimated_time`
   - `warning`

### Prompt 工程建议

- 提示词里明确要求“先排序，再拆分，再输出”
- 明确每天任务量不能失衡
- 如果输入信息不足，允许模型输出“需要补充的信息”
- 强制模型用 JSON 或字段化文本输出，方便前端展示

### 本节课不建议做的内容

- 自动手机监督
- 系统权限读取
- 小组件联动
- 用户长期历史学习建模

### 课堂验证建议

建议用 3 组样本测试：

1. 任务少但时间紧
2. 任务多且多个 DDL 冲突
3. 输入模糊、容易导致计划失衡的样本

重点验证：

- 计划是否可执行
- 任务拆分是否足够细
- 输出是否能直接给学生使用
